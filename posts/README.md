## 🎯 使用说明

### 配置文章目录

在 `config/common.yaml` 中设置你的文章目录：

```yaml
# 文章存放目录
content_dir: /path/to/your/articles/
```

### 文章格式要求

#### 标准格式
支持标准 Markdown 格式：

```markdown
# 文章标题

文章正文内容...
```

#### 推荐格式（包含元数据）
使用 Front Matter 添加文章元数据：

```markdown
---
title: 文章标题
description: 文章描述，用于摘要
tags:
  - Python
  - 机器学习
  - 深度学习
category: 技术分享
cover: https://example.com/cover.jpg
author: 作者名称
date: 2025-11-06
---

# 文章正文

这里是文章的正文内容...
```

## 🏷️ Front Matter 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | 字符串 | 是 | 文章标题 |
| `description` | 字符串 | 否 | 文章描述/摘要 |
| `tags` | 数组 | 否 | 文章标签列表 |
| `category` | 字符串 | 否 | 文章分类 |
| `cover` | URL | 否 | 封面图片链接 |
| `author` | 字符串 | 否 | 作者名称 |
| `date` | 日期 | 否 | 发布日期 |

## 📋 最佳实践

### 标题建议
- 长度控制在 15-30 字
- 包含关键词，便于搜索
- 避免使用特殊符号
- 体现文章价值

### 标签选择
- 每篇文章 3-5 个标签
- 选择相关性高的标签
- 使用平台热门标签
- 避免过于宽泛的标签

### 内容结构
```markdown
# 标题

## 概述
简要介绍文章主题和内容

## 核心内容
### 子主题1
具体内容...

### 子主题2  
具体内容...

## 实践案例
提供具体的代码或操作示例

## 总结
总结要点和收获
```

### 图片使用
- 使用稳定的图片链接
- 推荐使用图床服务
- 图片尺寸适中（宽度建议800-1200px）
- 添加合适的alt文本
